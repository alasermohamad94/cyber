## 1. نظرة عامة

نظام الدفاع السيبراني (Cyber Defense System — CDS) منصة لمراقبة الأمان وتحليل السلوك والاستجابة الآلية للتهديدات. يجمع بين:

- **محرك كشف قائم على القواعد** (Rule Engine) لتحليل السلوك والتنبؤ بمراحل الهجوم واتخاذ القرار.
- **استدلال ML ظلّي** (Shadow ML) استشاري لا يستبدل القواعد.
- **واجهة ويب عربية** للمراقبة التشغيلية وإدارة التهديدات.
- **تخزين دائم** للأحداث والسياسات في PostgreSQL (أو SQLite للتطوير المحلي).

### المكدس التقني

| الطبقة | التقنية |
|--------|---------|
| Backend | FastAPI + Uvicorn |
| Database | PostgreSQL |
| Frontend | React + TypeScript (Vite) |
| المصادقة | جلسات Starlette + RBAC |
| الوقت الفعلي | WebSocket (`/ws/metrics`) |

## 3. خط أنابيب الكشف والاستجابة

يمر كل كيان (جهاز، مستخدم، خادم) عبر خط أنابيب متسلسل:

```
بيانات خام (Telemetry)
        ↓
┌───────────────────┐
│  Perception       │  تحليل السلوك → درجة سلوك [0–100]
└─────────┬─────────┘
          ↓
┌───────────────────┐
│  Prediction       │  مرحلة الهجوم الحالية والتالية + ثقة
└─────────┬─────────┘
          ↓
┌───────────────────┐
│  ML Shadow        │  استشارة اختيارية (CDS_ML_SHADOW_MODE)
└─────────┬─────────┘
          ↓
┌───────────────────┐
│  Trust System     │  تحديث درجة الثقة الديناميكية
└─────────┬─────────┘
          ↓
┌───────────────────┐
│  Decision Engine  │  monitor | alert | block | isolate
└─────────┬─────────┘
          ↓
┌───────────────────┐
│  Response Engine  │  تنفيذ الإجراء وتسجيله
└───────────────────┘
```

### 3.1 طبقة الإدراك (`perception/`)

تحلل بيانات الكيان الخام:

| المدخل | الوصف |
|--------|--------|
| `connection_rate` | معدل الاتصالات |
| `request_rate` | معدل الطلبات |
| `failed_auth_count` | محاولات مصادقة فاشلة |
| `total_auth_count` | إجمالي محاولات المصادقة |
| `unique_ports` | منافذ فريدة (استطلاع) |
| `sensitive_access_count` | وصول لموارد حساسة |

**المخرج:** `BehaviorProfile` — درجة سلوك، مستوى شذوذ (`low` → `critical`).

### 3.2 طبقة التنبؤ (`prediction/`)

تربط السلوك بمراحل دورة حياة الهجوم:

`normal` → `reconnaissance` → `initial_access` → `lateral_movement` → `privilege_escalation` → `data_exfiltration`

**المخرج:** المرحلة الحالية، المرحلة المتوقعة تالياً، درجة الثقة.

**ML Shadow** (`prediction/model_inference.py`): نموذج استشاري يُفعَّل عبر `CDS_ML_SHADOW_MODE=true` ولا يغيّر القرار النهائي وحده.

### 3.3 محرك القرار (`decision_engine/`)

يجمع: درجة السلوك + مرحلة الهجوم + درجة الثقة (+ استشارة ML).

| الإجراء | شروط تقريبية |
|---------|----------------|
| `isolate` | سلوك ≥ 85 أو مراحل حرجة أو ثقة < 20 |
| `block` | سلوك ≥ 70 أو `lateral_movement` أو ثقة < 40 |
| `alert` | سلوك ≥ 50 أو `initial_access` أو ثقة < 60 |
| `monitor` | باقي الحالات |

### 3.4 نظام الثقة (`trust_system/`)

- درجة ابتدائية 100، تتناقص مع السلوك السيء وتتحسن مع الجيد.
- اضمحلال زمني (`decay_rate`)، سجل تاريخي محدود.
- مستويات مخاطر: `low`, `medium`, `high`, `critical`.

### 3.5 محرك الاستجابة (`response/`)

ينفّذ القرار: مراقبة معززة، تنبيه، حظر، عزل.  
يسجّل كل إجراء في قاعدة البيانات مع الحالة والتوقيت.

## 4. الواجهة الخلفية (FastAPI)

### 4.1 التشغيل

```bash
pip install -r requirements.txt
python -m backend.main
# أو: uvicorn backend.main:app --host 127.0.0.1 --port 8080
```

### 4.2 واجهات API الرئيسية

| المسار | الصلاحية | الوظيفة |
|--------|----------|---------|
| `POST /api/login` | عام | تسجيل الدخول |
| `GET /api/session-info` | مصادق | الجلسة والصلاحيات |
| `GET /api/system-metrics` | `metrics:read` | CPU، ذاكرة، قرص، شبكة |
| `GET /api/security-overview` | `security:read` | ملخص أمني |
| `GET /api/analytics-overview` | `metrics:read` | تحليلات موحّدة |
| `GET /api/threat-overview` | `security:read` | تهديدات وIPs |
| `POST /api/analyze-entity` | `entity:analyze` | تحليل كيان عبر المحرك |
| `POST /api/block-ip` | `ip:block` | حظر عنوان IP |
| `POST /api/unblock-ip` | `ip:unblock` | إلغاء الحظر |
| `GET /api/security-report` | `report:generate` | تقرير JSON |
| `GET /api/health` | عام | فحص صحة الخدمة |
| `WS /ws/metrics` | مصادق | بث مقاييس لحظية |

### 4.3 عقد المقاييس

ملف `backend/metrics_contract.py` يعرّف مصدر كل مؤشر (psutil، trust_system، security_store…) ويُرفق `data_quality` مع كل استجابة API.

## 7. الأمان والصلاحيات (RBAC)

### الأدوار

| الدور | الوصف |
|-------|--------|
| `admin` | كامل الصلاحيات بما فيها حظر IP والإعدادات |
| `analyst` | تحليل كيانات وتقارير — بدون حظر IP |
| `viewer` | قراءة المقاييس والأمان فقط |

### الصلاحيات

| الصلاحية | admin | analyst | viewer |
|----------|:-----:|:-------:|:------:|
| `metrics:read` | ✓ | ✓ | ✓ |
| `security:read` | ✓ | ✓ | ✓ |
| `entity:analyze` | ✓ | ✓ | — |
| `report:generate` | ✓ | ✓ | — |
| `ip:block` / `ip:unblock` | ✓ | — | — |
| `settings:write` | ✓ | — | — |

### متغيرات البيئة (`.env`)

| المتغير | الوصف |
|---------|--------|
| `CDS_SECRET_KEY` | مفتاح الجلسة |
| `CDS_ADMIN_USER` / `CDS_ADMIN_PASSWORD` | حساب المدير |
| `CDS_ANALYST_*` / `CDS_VIEWER_*` | حسابات أخرى |
| `CDS_DATABASE_URL` | اتصال PostgreSQL |
| `CDS_CORS_ORIGINS` | أصول الواجهة المسموحة |
| `CDS_FIREWALL_ENABLED` | تطبيق قواعد netsh/iptables |
| `CDS_ML_SHADOW_MODE` | تفعيل ML الاستشاري |

## 11. القيود والاعتبارات

1. **جدار الحماية:** يحاول تطبيق `netsh` (Windows) أو `iptables` (Linux) — قد يتطلب صلاحيات مسؤول.
2. **ML Shadow:** استشاري فقط؛ القرار النهائي من Rule Engine.
3. **PostgreSQL:** مطلوب للإنتاج؛ SQLite للتطوير السريع فقط.
4. **الجلسات:** تعتمد على `CDS_SECRET_KEY` ثابت في الإنتاج.
