## 2. المتغيرات — ما تم تغييره في المشروع

### 2.1 أمان وتشغيل (P0)

| المتغير | الحالة قبل | الحالة بعد | الملفات |
|---------|------------|------------|---------|
| مصادقة الواجهة | لا يوجد | جلسة Flask + `/login` `/logout` | `security/auth.py`, `login.html` |
| تفويض الأدوار | لا يوجد | admin / analyst / viewer + `@require_permission` | `security/roles.py`, `app.py` |
| مصدر الدور | جلسة قابلة للتلاعب | **`get_current_role()`** يعيد الدور من قاموس المستخدمين | `security/auth.py` |
| مفتاح الجلسة | ثابت | `CDS_SECRET_KEY` | `security/config.py` |
| ربط الشبكة | `0.0.0.0` + CORS `*` | `CDS_BIND_HOST` + `CDS_CORS_ORIGINS` | `app.py` |
| حظر IP | تسجيل فقط | netsh/iptables + SQLite | `security/firewall.py` |
| Socket.IO | eventlet | `async_mode="threading"` | `app.py` |
| تحميل إعدادات | يدوي | **`.env`** تلقائي من جذر المشروع | `security/config.py` |
| أسماء مستخدمين مكررة | تطغى صامتاً | **رفض** مع `ValueError` واضح | `security/config.py` |

**صلاحيات الأدوار (مُنفَّذة API + UI):**

| الدور | metrics/security | entity:analyze | ip:block/unblock | report | settings:write |
|------|------------------|----------------|------------------|--------|----------------|
| admin | ✅ | ✅ | ✅ | ✅ | ✅ |
| analyst | ✅ | ✅ | ❌ | ✅ | ❌ |
| viewer | ✅ | ❌ | ❌ | ❌ | ❌ |

**آليات RBAC في الواجهة (2026-06):**

- `@require_permission` على مسارات `/api/*` الحساسة
- `cds_has_perm()` في قوالب Jinja — إخفاء الأزرار من الخادم
- `body.cds-role-viewer` / `cds-role-analyst` في CSS
- `cds-app.js` — `data-permission` + `CDS.requirePermission()`
- `/api/session-info` — `role` + `permissions[]`
