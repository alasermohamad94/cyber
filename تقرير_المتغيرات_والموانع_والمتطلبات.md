## 2. المتغيرات — ما تم تغييره في المشروع

### 2.1 أمان وتشغيل (P0)

| المتغير | الحالة قبل | الحالة بعد | الملفات |
|---------|------------|------------|---------|
| مصادقة الواجهة | لا يوجد | جلسة Flask + `/login` `/logout` | `security/auth.py`, `login.html` |
| تفويض الأدوار (AuthZ) | لا يوجد | admin / analyst / viewer + `@require_permission` | `security/roles.py`, `app.py` |

### 2.3 متانة النظام (P2)

| المتغير | التفاصيل |

### 2.4 ذكاء اصطناعي (P3 — جزئي)

| المتغير | التفاصيل |
|---------|----------|
| دمج ML في المسار | `main.py` → `shadow_predict()` بعد perception |
| النموذج | Isolation Forest (scikit-learn) مع fallback heuristic |

| القرار | `decision_engine` يدمج `ml_advisory` في reasoning وconfidence |

## 3. الموانع — ما منع الإكمال الكامل

الموانع هي عوائق **لا يمكن تجاوزها بالكود فقط** دون تدخل بيئي أو تشغيلي أو قرار إداري.

| # | المانع | السبب | ما تم بديله | ما يلزم لإزالة المانع |
|---|--------|-------|-------------|------------------------|
| 1 | **ML إنتاجي كامل** | لا بيانات موسومة ولا تيليمتري تاريخي للتدريب | Isolation Forest baseline + Shadow | جمع incidents + labels + pipeline تدريب |
| 2 | **جدار ناري فعلي** | netsh/iptables يحتاج Administrator/root | تسجيل في DB + `firewall_applied` | تشغيل كمسؤول أو API لـ NGFW |
| 3 | **طوابير موزعة** | لا Redis/RabbitMQ/Celery | `background_queue` داخل العملية | نشر broker + workers منفصلين |
| 4 | **حذف CLI dashboards** | خطر كسر سكربتات قديمة | `DEPRECATED.md` فقط | قرار إداري + إصدار major |
| 5 | **هوية مؤسسية** | لا LDAP/OAuth/SAML مهيأ | 3 مستخدمين محليين | ربط IdP + MFA |
| 6 | **اختبارات حمل** | لا CI حمل لـ Socket.IO | اختبارات وحدة/تكامل أساسية | Locust + CI متعدد العمليات |

## 4. المتطلبات التي لم تتوفر — مقابل تقرير التصحيح والتطوير

هذا القسم يربط **متطلبات PDF** (القسم 2 — خطة التطوير) بما **لم يُوفَّر** أو **لم يُنفَّذ**.

### 4.1 متطلبات بيانات (Data Requirements)

| المتطلب (PDF) | متوفر؟ | التأثير |
|---------------|---------|---------|
| Telemetry زمني (host/process/network/auth) | **جزئي** — psutil + entity fields في التحليل فقط | لا سجل تيليمتري مركزي طويل الأمد |
| Labels لأحداث مؤكدة (incident outcomes) | **لا** | لا تدريب XGBoost/LightGBM |
| بيانات سياقية (حساسية الأصل، baseline تاريخي) | **لا** | لا مقارنة entity baseline دقيقة |
| Feature engineering (rolling, entropy, burst) | **جزئي** — vector بسيط في ML | لا pipeline features كامل |

---

### 4.2 متطلبات معمارية

| المتطلب (PDF) | متوفر؟ | الواقع الحالي |
|---------------|---------|----------------|
| فصل طبقات ingestion / detection / decision / response / audit | **جزئي** | modules منفصلة؛ لا حدود خدمات صارمة |
| مخزن حالة دائم | **نعم** | SQLite |
| workers/queues للعمليات الثقيلة | **جزئي** | in-process queue فقط |
| إزالة ازدواجية لوحات `dashboard/` | **لا** | إهمال فقط |
| Metric Contract موحّد | **نعم** | `metrics_contract.py` |
| منع KPI بدون API | **نعم** للصفحات المحدّثة | |
| فصل Demo / Production بصرياً | **جزئي** | flag + disclaimer API |
| توحيد أرقام الصفحات الثلاث | **محسّن** | مصادر API موحدة |
| مؤشرات جودة بيانات | **نعم** | `data_quality` |

---

### 4.3 متطلبات الذكاء الاصطناعي (P3)

| المتطلب (PDF) | متوفر؟ | ملاحظة |
|---------------|---------|--------|
| كشف شذوذ (Isolation Forest / SVM / Autoencoder) | **جزئي** | Isolation Forest على baseline اصطناعي |
| نمذجة سلسلة زمنية (LSTM / Temporal CNN) | **لا** | لا بيانات زمنية كافية |
| تصنيف هجومي مراقب (XGBoost/LightGBM) | **لا** | لا labels |
| model_inference بعد perception وقبل decision | **نعم** | في `main.py` |
| دمج score قواعد + نموذج في policy | **جزئي** | Canary للثقة فقط |
| Shadow Mode | **نعم** | افتراضي |
| Canary rollout | **جزئي** | `CDS_ML_CANARY` — ثقة فقط |
| تعميم تدريجي + مراقبة drift | **لا** | يحتاج إنتاج ML |
